Ecosystem ES6 Introduction - Arrow Functions

Arrow functions ( => ) are yet another feature which appeared on many languages such as scala, more recently java, and probably many more. But what does it add to Javascript, since anonymous functions can already be used for callbacks?

  • A lighter syntax
  • There is not a specific this attached to them, which can be inrteresting when working in classes for example (no more mix between the this of the class instance and the one of the current function)
// class Person{ constructor(firstName, lastName, age, friends){ this.firstName= firstName; this.lastName= lastName; this.age= age; this.friends= friends; this.computeMaxAgeFromFriends(); } // Only one instruction, no block, // then the return keyword is implicit getfirstName= () => this.firstName; // I can use a block as well getLastName= () => { return this.lastName; } //No specific this is bound to arrow functions, // so this always refer to the class instance, which // would not be the case with classic functions let computeMaxAgeFromFriends= () => { this.maxAge= 0; this.friends.forEach( (friend) => { if (friend.age > this.maxAge){ this.maxAge= friend.age; } } ); } }
