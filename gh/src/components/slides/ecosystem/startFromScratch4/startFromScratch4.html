Ecosystem Start From Scratch - polyfills and libs

Polyfills are scripts which improve browsers abilities. For example, if you decide to use Babel to transpile your ES6 at runtime, you're using a polyfill.

typescript allows you to choose the syntax you want for modules (AMD, CommonJS and some others). Once generated, your browser will need a polyfill to understand this syntax. Using SystemJs is a common way to do this

Angular2 allows you to work with reactive programming, but actually doesn't include its own implementation. In order to do so, you will have to use Rx.js

Finally, a minimal main html page imports could look like this (zone.js and reflect.js features are provided by angular2-polyfills.js)

<script src="https://code.angularjs.org/2.0.0-beta.0/angular2-polyfills.js"></script> <script src="https://code.angularjs.org/tools/system.js"></script> <script src="https://code.angularjs.org/2.0.0-beta.0/Rx.js"></script>
