Main Concepts Router - Configuration And Usage

Lot of things in the previous example

  • Firstly, we have to load our router library this is the aim of router.dev.js
  • We use the @RouteConfig decorator to tell which component has to be displayed for a given url
  • We then import some needed types both for use and configure the router:
    • ROUTER_PROVIDERS: Which contains needed providers we have to provide to the app
    • ROUTER_DIRECTIVES: To specify that our component will need such directives
    • LocationStrategy and HashLocationStrategy to specify the way urls will be managed by the router this way: server:port/app/#/your/path
    • HashLocationStratefy is not the default value. By default, urls are managed with the pushState api and urls looks like: server:port/app/your/path. More details here
  • Finally we use our directives:
    • the routerLink to go to a view whose the name comes from our @RouteConfig declaration
    • And router-outlet to specify where our view will be loaded
