Main Concepts Router - Configuration And Usage

Things to see in the previous example

  • The @RouteConfig annotation to map an url to a Component
  • Several imports to configure the router
    • ROUTER_PROVIDERS: Which contains needed providers we have to provide to the app
    • ROUTER_DIRECTIVES: is needed for components which will use the router directives
    • LocationStrategy and HashLocationStrategy to specify the way urls will be managed by the router, in our case, thiw way: server:port/app/#/your/path
    • HashLocationStrategy is not the default value. By default, urls are managed with the pushState api and urls looks like: server:port/app/your/path. More details here
  • Finally we use our directives:
    • the routerLink to go to a view whose the name comes from our @RouteConfig declaration
    • And router-outlet to specify where our view will be loaded
