Main Concepts Router - Child Routers

We are almost done with the Router api:

  • We can create a routing Component to define our routes
  • Navigate to these routes both dynamically and declaratively
  • Check that we succeeded going to this view
  • And pass parameters when switching of view

We miss one important concept, the Child Component Routing which allows to manage a navigation feature in any child component of your application, say independently of your parent component routing. Don't forget that your components are such kind of Web Components and that they should ideally be reusable somewhere else
