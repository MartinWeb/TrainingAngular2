Main Concepts About Templates - Event Binding

Listening events is a common need in javascript, and this was easy enough to do it with AngularJs. In order to do so and listen to the click event for example, we had to use the ngClick directive.

  • Disadvantage: one directive for each event means a new event needs a new directive
  • with Angular2, no use of directive but parenthesis
  • It's a native behavior, and you don't need any new devlopment to manage new events
  • $event is made available in the associated expression if you need
import {bootstrap} from 'angular2/platform/browser'; import {AppComponent} from 'app/components/app'; bootstrap(AppComponent, []); import {Component} from '@angular/core'; @Component({ selector: 'my-app', templateUrl: 'app/components/app.html' }) export class AppComponent { logEvent(msg, event){ console.log(msg); console.log(event); } } <h1>Checking event handling</h1> <p (click)="logEvent('click', $event)" > Click Me And check the logs </p>
