Ecosystem ES6 Introduction - Setting default parameters values

Javascript in general is a very permissive language, and allows to do stuff like:

  • to define functions with parameters, but call it without these parameters
  • Oppositely, to not define any parameters in your function but to access them if the function is called with parameters (with the arguments keyword)

ES6 now allow to define default values for parameters

//The old way var defaultParamValues1= function(param1, param2){ var param1Val= param1 || 'defaultVal1'; var param2Val= param2 || 'defaultVal2'; console.log(param1Val + ', ' + param2Val); }; defaultParamValues1(); defaultParamValues1('custom val1', 'custom val2'); //The new way var defaultParamValues2= function(param1 = 'defaultVal1', param2= 'defaultVal2'){ console.log(param1 + ', ' + param2); }; defaultParamValues2(); defaultParamValues2('custom val1', 'custom val2'); //For information only let getParams= function(){ console.log('I\'m called with the following arguments: ' + arguments); }

Sure, this is a lot more readable and so easier to maintain... And this remain very flexible, default values can be the result of a function call as well as other parameters of the current function call.

//This is valid var defaultParamValues1= function(param1 = 'firstParam', param2 = 'after ' + param1, param3 = getValue(param2)){ console.log(param1 + ', ' + param2 + ', ' + param3); };

Finally, this also work with destructuring assignment

let person= {firstName:'bob'}; let defaultToMarley= {firstName, lastName='Marley'} = person;
