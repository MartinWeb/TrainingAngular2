Preamble Angular2 VS Polymer VS React - What's the point

Angular2 VS Polymer Js

  • Polymer is not really a framework, unlike Angular2
  • Polymer gives you a set of components and allows anybody to import them in its own application
  • So Polymer is not really done to do SPAs, with stuff like routing and lifecycles features...
  • It's more a library to make Web Components and can be used for example with Meteor Js in order to add framework features
  • Finally, Polymer components can be used inside Angular2 applications as well...

Angular2 vs React

  • Common points
    • Both are frameworks which allow to define view applications
    • Both provide ways to choose the view rendering: browser/ desktop / mobile
    • Checker les vuews desktop pour angular2
    • Both are based on Web Components
    • Both allow you to work with paradigms/patterns such as FRP (Functional Reactive Programming), OOP (Object Oriented Programming) & flux
  • Main differences
    • A React application is a set of libraries provided by the community that the developer has to choose and assemble in order to get all functionalities he needs: More flexible but more work and less stable (be sure that your libraries will be maintain, readapt best practices for a new project...)
    • Angular2 core library contains everything needed to make a complete application work
    • Angular2 provides an Inversion Of Control pattern (or Dependency Injection) associated with specific components such as Pipes, Services (...) for a great separation of concerns
    • The mechanisms to detect changes are different, the one of Angular2 can be redefined
      • React: Automatic Virtual DOM differences detection plus custom implementations to disable rendering of some components. The Virtual Dom is an "in memory" version of the DOM, once it's state change, an algorithm compares it with the real DOM and find the faster way to update the real DOM (so with a minimal number of operation)
      • Angular: On each browser event (js turn), the detection change algorithm is called to check each component changes (details here) and can be strongly faster with good practices such as work with immutable objects
    • React is a little easier to learn
