Main Concepts About Templates - Property Binding - Considerations

Expressions are almost like js, but are not:

  • The context is the bean ( access to the bean properties, not global js variables )
  • The new operator is prohibited
  • As wel as increment / decrement operators
  • Bitwise operators ( | and & ) are not supported
  • Use filters in expressions with |

As in Angular1, remember that expressions are in the view, and that the view shouldn't contains complex code. Then keep expressions simple and prefer add a method on your bean. 

    //NO
    [isEligible]="(user.age > 13 && user.friends.length > 3 && isInList(user.preferedMovie)"

    //OK
    [isEligible]="checkEligible()"

If your HTML parser doesn't like brackets, use the canonical syntax: 

    bind-property-name="your expression"
