Main Concepts Router - Configuration And Usage

So to declare our routes, we can use @RouteConfig

  • Which takes in parameters an array of RouteDefinition objects
  • A RouteDefinition is an interface: export interface RouteDefinition { path?: string; aux?: string; regex?: string; serializer?: RegexSerializer; component?: Type | ComponentDefinition; loader?: () => Promise<Type>; redirectTo?: any[]; as?: string; name?: string; data?: any; useAsDefault?: boolean; }
  • Whose Route is the most common implementation and which needs the following parameters:
    • The route path: the url in the browser
    • The route name: an identifier for this route
    • A component type: the component to load

If, for some reason, you need to define your routes programmatically, then use the Router service and its method config which takes the same parameters than @RouteConfig
