Main Concepts Dependency Injection - Introduction

  • DI in Angular2 will mainly consist in injecting Services into Components, Pipes or other Services
  • The basic DI mechanism is based on types only, we can't do easier.
import {LogLevel} from '/path/to/LogLevel'; export class LoggerService { private minLevel: LogLevel = LogLevel.WARN info(msg: String): void{ if (this.minLevel >= LogLevel.INFO){ console.log('[INFO]' + msg); } } warn(msg: String): void{ if (this.minLevel >= LogLevel.WARN){ console.log('[WARN]' + msg); } } error(msg: String): void{ console.log('[WARN]' + msg); } } import {LoggerService} from 'path/to/loggerservice'; // ... // ... export class PersonService{ constructor(private loggerService: LoggerService, private http: Http){} getPersonDetails= (person: Person):Promise<Person> => { return this.http.get('/api/person/' + person.getId()) .catch( (e: Exception): void => { this.loggerService.error(e.toString()); }); }; }

Ajouter la déclaration du service dans l'initialisation Vérifier si le catch peut s'utiliser comme ça.

Once the typescript is generated into ES5 metadata tags are used in order to describe types needed in order to instanciate a class. These metadata are used by Angular2 in order to know which dependencies hee needs. But this is a basic example, you will want to do much more sometimes.
