Main Concepts About Templates - Property Binding - the basics

  • Brackets are there to solve previous issues
  • These are not regular attributes, so they can have a custom behavior without breaking HTML5 specs
  • The value associated is evaluated as an expression
import {Component} from '@angular/core'; import {bootstrap} from 'angular2/platform/browser'; import {SubCmp} from './sub-cmp'; @Component({ selector: 'my-app', template: '<sub-cmp name="john" [friends]="friends" ></sub-cmp>', directives: [SubCmp] }) export class AppComponent { name: string= 'john'; friends: string[]= ['bob', 'dylan']; } bootstrap(AppComponent), []); import {Component, Input} from '@angular/core'; @Component({ selector: 'sub-cmp', template: ` <h1> Name: {{name}}</h1> <p>Friends: </p> <ul> <li *ngFor="#friend of friends">{{friend}}</li> </ul>` }) export class SubCmp { @Input() name: string; @Input() friends: string[]; }
  • Do we have the same array instance (friends) AppComponent and SubCmp ?
