Main Concepts Classes Are Everywhere - About Services - Router

  • Router allows you to manage routes in your application, which means basically
    • Associate components to pathes (urls)
    • Access to these components when this url is used
    • Access to these components programmatically
  • Again, the idea here is to give you the basics, let's get more details later, because routing is a complete feature, not a single service
  • For now, the example below shows you basics only:
    • How to configure routes
    • How to use directives to navigate to these routes
//Some router dependencies import {RouteConfig, ROUTER_DIRECTIVES} from '@angular/router-deprecated'; import {TodoListCmp} from 'todolist.ts'; import {TodoFormCmp} from 'todoform.ts'; // You can configure your router at the component level (basically on the root one, but not only...) // Use the @RouteConfig annotation @RouteConfig([ {path:'/todolist', name:'TodoList', component:TodoListCmp}, {path:'/todoform', name:'TodoForm', component:TodoFormCmp} ]) @Component({ selector:'app', templateUrl:'app.html', directives: [ROUTER_DIRECTIVES] }) export class App {} <nav> <ul> <li><a [routerLink]="['TodoList']">Go to Todo List</a></li> <li><a [routerLink]="['TodoForm']">Go to Todo Form</a></li> </ul> </nav> <router-outlet></router-outlet>
