But typescript doesn't create these metadata for every class. Yes, why would it do that? typescript is not related only to Angular2, and even in this case, every class doesn't necessarily need a DI mechanism.

In fact, typescript will generate these metadata when Annotations are associated to a class or its constructor. Which means that they will be added for components (@Component), pipes (@Pipe), but not for services which, by default are not annotated.

To force metadata to be generated also for services, you can add an annotation on this service, 2 solutions:

  • Add @Inject on the cpnstructor parameters: 
    constructor(@Inject(MyService) myService: MyService){}
  • @Injectable on the service class: 
                @Injectable
            class MyService{
                constructor(dep: MyDepService){ 
                    //...
                }
            }
